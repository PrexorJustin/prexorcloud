| **Affinity Label** | Key-value pair on a node that a group requires for placement. Instances with affinity labels will only be placed on matching nodes. | `GroupConfig`, `WeightedNodeSelector` |
| **Anti-Affinity Label** | Key-value pair on a node that a group rejects. Instances will not be placed on nodes with matching anti-affinity labels. | `GroupConfig`, `WeightedNodeSelector` |
| **Audit Log** | Chronological record of state-changing API operations (who, what, when, resource type). | `RestServer.audit()`, `AuditRepository` |
| **Bootstrap** | The one-time process where a daemon exchanges a join token for mTLS certificates. | `BootstrapManager`, `BootstrapServiceImpl` |
| **CA (Certificate Authority)** | Self-managed PKI that issues TLS certificates for controller and daemon nodes. | `CertificateAuthority` |
| **Catalog** | Registry of available platform JARs (e.g., Paper 1.21.4, Velocity 3.4.0). Maps platform + version to download URL. | `CatalogStore`, `CatalogRoutes` |
| **CDS (Class Data Sharing)** | JVM optimization that pre-computes class metadata for faster startup. Generated by `PaperBootstrapCache`. | `PaperBootstrapCache` |
| **ClusterState** | In-memory authoritative model of all nodes, instances, players, and metrics. Thread-safe via ConcurrentHashMap. | `ClusterState` |
| **ClusterView** | Read-only snapshot of cluster state, exposed via the public API to modules and plugins. | `ClusterView`, `ClusterStateAdapter` |
| **Console Buffer** | Ring buffer storing recent console output lines per instance. Used for SSE streaming. | `ConsoleBuffer`, `ConsoleStreamer` |
| **Controller** | Central orchestration server managing cluster state, scheduling, REST API, and gRPC services. | `PrexorController`, `PrexorCloudBootstrap` |
| **Cooldown** | Minimum time between scaling actions for a group, preventing flapping. | `ScalingEvaluator` |
| **Crash Loop** | When a group's instances crash repeatedly within a sliding window. Triggers automatic group pause. | `CrashLoopDetector` |
| **Daemon** | Node agent process that connects to the controller, manages server processes, and monitors resources. | `PrexorDaemon` |
| **Deployment** | A rolling restart operation tracked with progress, status, and metadata. | `DeploymentRecord`, `Scheduler.rollingRestart()` |
| **Drain** | Graceful node evacuation: stop accepting new instances, migrate players, stop remaining instances. | `NodeDrainManager`, `NodeRoutes` |
| **Dynamic Scaling** | Scaling mode that automatically adjusts instance count based on player load thresholds. | `ScalingEvaluator` |
| **EventBus** | Pub-sub event system with class-based, type-string, and wildcard channels. | `EventBus` (controller), `EventBus` (API) |
| **Forwarding Secret** | Shared secret between proxy and server plugins for player forwarding verification. | `ServerConfigPatcher` |
| **Group** | Logical collection of instances sharing the same configuration (platform, templates, scaling rules). | `GroupConfig`, `GroupManager` |
| **Handshake** | Initial gRPC message exchange when a daemon connects to the controller. | `DaemonGrpcClient`, `DaemonServiceImpl` |
| **Heartbeat** | Periodic ping/pong between controller and daemon to detect connection loss. | `HeartbeatTracker`, `HeartbeatScheduler` |
| **Instance** | A single Minecraft server or proxy process managed by a daemon. | `InstanceInfo`, `ServerProcess` |
| **Instance State** | Lifecycle state: SCHEDULED â†’ PREPARING â†’ STARTING â†’ RUNNING â†’ STOPPING â†’ STOPPED/CRASHED. | `InstanceState` |
| **Join Token** | One-time credential for daemon registration. Exchanged for mTLS certificates via Bootstrap RPC. | `JoinToken`, `FileJoinTokenStore` |
| **Maintenance Mode** | Global or per-group flag that prevents new instance scheduling. | `MaintenanceConfig`, `Scheduler` |
| **MiniMessage** | Text formatting syntax for Minecraft chat messages and MOTDs (similar to HTML). | `minimessage.ts` |
| **Module** | Controller-side extension with REST routes, event handlers, data stores, and optional frontend. | `CloudModuleBase`, `ModuleLoader` |
| **ModuleDataStore** | MongoDB-backed per-module data store with `mod_{name}_` collection prefix isolation. | `ModuleDataStore`, `MongoStateStore` |
| **mTLS** | Mutual TLS â€” both client and server present certificates for authentication. | `ServerTlsCredentials`, `ClientTlsCredentials` |
| **Node** | A physical or virtual machine running a daemon agent. | `NodeState`, `NodeSession` |
| **NodeSelector** | Strategy interface for choosing which node to place an instance on. | `NodeSelector`, `WeightedNodeSelector` |
| **NodeSession** | Per-node gRPC stream handle for bidirectional communication. | `NodeSession`, `NodeSessionManager` |
| **Plugin** | Server/proxy-side extension running inside Minecraft JVM. Uses CloudAPI for commands, events, players. | `CloudPluginBase`, `CloudPluginContext` |
| **Plugin Token** | Per-instance bearer token (`ptk_` prefix) for plugin â†’ controller REST communication. | `ClusterState.pluginTokens` |
| **Port Allocation** | Sequential scan of configured port range to find the first unused port on a node. | `PortAllocator` |
| **Pre-Warm** | Proactive cache warming: downloading JARs and generating CDS before instances are needed. | `PreWarmService`, `PaperBootstrapCache` |
| **RBAC** | Role-Based Access Control with 28+ granular permissions grouped by resource type. | `Role`, `Permission`, `JwtAuthMiddleware` |
| **Reconciliation** | On daemon reconnect, comparing controller's known instances against daemon's running list. Missing instances are marked CRASHED. | `DaemonServiceImpl.handleHandshake()` |
| **RedisEventBridge** | Optional Redis pub/sub bridge enabling event distribution across multiple controllers. | `RedisEventBridge` |
| **Rolling Restart** | Sequential instance restart with replacement waiting, ensuring zero-downtime deployments. | `Scheduler.rollingRestart()` |
| **Scaling Mode** | How a group manages instance count: STATIC (fixed), DYNAMIC (auto-scale), MANUAL (operator-driven). | `GroupConfig.scalingMode` |
| **Scheduler** | Periodic evaluation loop that places and removes instances based on group configuration. | `Scheduler` |
| **SSE (Server-Sent Events)** | HTTP-based push protocol used for real-time event streaming to dashboards. | `SseEventStreamer`, `ConsoleStreamer` |
| **StateStore** | Persistence abstraction for durable data (MongoDB implementation). | `StateStore`, `MongoStateStore` |
| **Static Scaling** | Scaling mode that maintains a fixed set of instances with deterministic IDs. | `Scheduler.evaluateGroup()` |
| **Template** | A set of files (configs, plugins, worlds) applied to instances via layered composition. | `TemplateManager`, `TemplateMerger` |
| **Template Chain** | Ordered template list: base â†’ base-{platform} â†’ {group} â†’ user templates. Each layer overlays files. | `Scheduler.doPlaceInstance()` |
| **Topological Sort** | Kahn's algorithm used to order groups by `dependsOn` relationships during scheduling. | `Scheduler.evaluate()` |
| **Variable Substitution** | Replacing `{PORT}`, `{INSTANCE_ID}`, etc. in template config files with actual values. | `VariableSubstitution` |
| **Version Adaptation** | Plugin pattern enabling a single JAR to support multiple Minecraft versions via `@ForVersion` annotations. | `CloudPluginBase.adapt()`, `VersionDispatcher` |
| **Webhook** | HTTP POST notification sent to external URLs on specific events (crashes, disconnections). | `WebhookAlertService` |
