# prexorctl Command Catalog

`prexorctl` is the command-line client for PrexorCloud. It is a [cobra](https://github.com/spf13/cobra)-based CLI that talks to a controller over its REST API (`/api/v1/*`) and renders results either as styled terminal output (tables, cards, sparklines, pickers, and full-screen log/console views built with the [charmbracelet](https://github.com/charmbracelet) stack — bubbletea, lipgloss, huh) or, with `--json`, as machine-readable JSON. It also performs local-only work (scaffolding modules and plugins, validating jars, running gradle, collecting diagnostics) and host lifecycle work (installing and stopping the controller, daemon, and dashboard). This document is a neutral, complete reference of every command and subcommand for use in a CLI redesign.

---

## Global flags & environment

### Persistent flags (apply to every command)

| Flag | Short | Type | Default | Effect |
|------|-------|------|---------|--------|
| `--json` | `-j` | bool | false | Emit JSON instead of styled output. Also forces no-color and disables interactive pickers. |
| `--controller` | `-c` | string | "" | Override the controller URL for this invocation. |
| `--token` | `-t` | string | "" | Override the auth token for this invocation. |
| `--context` | — | string | "" | Override the active context for this invocation. |
| `--no-color` | — | bool | false | Disable ANSI color. |
| `--ascii` | — | bool | false | ASCII-only glyphs (no unicode box drawing, sparklines, pills). |
| `--verbose` | `-v` | bool | false | Print HTTP request/response trace lines (`→`/`←`). |

### Environment variables

| Variable | Effect |
|----------|--------|
| `PREXOR_OUTPUT=json` | Sets `--json` at startup. |
| `PREXOR_CONTROLLER` | Controller URL. |
| `PREXOR_TOKEN` | Auth token. |
| `PREXOR_CONTEXT` | Active context name. |
| `PREXOR_NO_BROWSER`, `CI` | Suppress the browser open in `setup`. |
| `NO_COLOR` | Forces no-color, independent of `--no-color`. |

Resolution precedence for controller / token / context: **flag > `PREXOR_*` env > stored context**.

Configuration is stored at `~/.prexorcloud/config.yml` (file mode 0600, dir 0700): `currentContext`, `contexts{name:{controller, token}}`, and `accent`. The token is a plaintext JWT.

### Exit codes

| Code | Meaning |
|------|---------|
| 0 | Success |
| 1 | Generic error |
| 2 | Auth error (HTTP 401) / diagnostic-warning convention |
| 3 | Forbidden (HTTP 403) |
| 4 | Not found (HTTP 404) |
| 5 | Connection error |

### Pre-link command gate

Before a controller context is resolved (no stored context and no `--controller`/env), only these top-level commands run: `setup`, `login`, `logout`, `version`, `help`, `completion`, `context`, `cluster`. Any other top-level command errors with "no cluster connected". Commands annotated `local-only` (e.g. `module new`/`scaffold`) skip the gate; in practice the purely local `plugin`, `module doctor`, and `module test` are still gated because only `module new` carries the annotation. The gate is skipped for `--help`.

### Shell completion

Static completions are generated by `prexorctl completion <shell>` for bash, zsh, fish, and powershell. Dynamic (resource-aware) completions are registered for groups, nodes, instances, crashes, contexts, and catalog platform/version arguments; they query the controller with a short timeout and fail silently when unconfigured. Dynamic completion works in zsh/fish/powershell; bash uses v1 generation, which does not call back into the binary.

## Reading the target dispositions

Each command below may carry a **Disposition** tag with one of: `KEEP` (default; usually omitted), `DECLUTTER` (remove or merge — reduce surface), or `CHANGE` (improve/extend the existing command). The tag records the *target* shape for the CLI redesign only; it never alters any factual field above it. Rationale, priority (P0/P1/P2), and the supporting audit live in **`cli-design-review.md`** — consult it before acting on any tag.

### Cross-cutting changes (apply CLI-wide)

See **`cli-modernization-roadmap.md`** for the phased plan + rationale; per-command detail in `cli/.audit/deep/01..07-*.md`.

- **Unified output system.** Replace the boolean `--json` with `-o {table,json,yaml,wide,name,jsonpath,template}` through one renderer (`--json` stays a hidden alias). Make output **total** — reads, mutations, *and* a structured JSON **error envelope**; `status --json` must stop being a strict subset of the TUI.
- **Async-convergence.** Add `--wait` / `--for=condition=running|deleted|healthy` / `--timeout` to every async op (`group scale`, `instance start/stop`, `node drain`, `group create`, `deploy <group>`, `cluster leave/join`), exit code by terminal state. They return before convergence today.
- **One danger-gate idiom.** A single helper: TTY→confirm, `--yes` bypass, typed-name confirm for quorum/blast-radius ops, `--dry-run` where sensible — on *every* destructive command (replaces today's mix of huh-confirm, raw `[y/N]` that silently no-ops in CI, typed-"yes", and no gate at all).
- **HA-awareness.** Multi-endpoint contexts + transparent any-member failover (free: shared JWT + `/cluster/members` restAddrs); surface leader/role/health/quorum in `cluster status`/`members`; make `logs`/`stop` member-aware.
- **Identity & session.** Real `whoami`/`auth status` (local JWT decode), token-expiry warnings + `auth refresh`, server-side `logout` revocation, scriptable `login` + CI service tokens.
- **No fabricated data.** Remove synthetic metrics in `status` (TPS/Players/Memory sparklines, `SPARK (1h)`) and `group list` (`UPDATED="just now"`, invented events, dead columns) — wire real values or omit.
- **Fix overloaded exit codes.** Give auth-401 its own code distinct from the diagnostic-warning `2`, and wire up the dead `ExitConnError=5`.
- **Secrets off argv.** No token/password as a positional/flag value (`config set token`, `context add --token`, `setup --*-password/-token`) — read from stdin/env.
- **URL-escape all id/name path args**; decode typed structs rather than `map[string]any`.
- **`completion bash` → `GenBashCompletionV2`** (dynamic completions are dead in bash today); complete enum + resource args/pickers across the board.
- **Unify theming** so huh prompts and `--help` honor `cfg.Accent` / `--no-color` / `--ascii` (they hardcode purple/v1 today); route spinners to stderr.

## setup

### `prexorctl setup`
- **Path**: `prexorctl setup [flags]`
- **Purpose**: Install/configure the Controller, Daemon, or Dashboard.
- **Args**: none (component is selected via `--component`, not a positional/subcommand).
- **Flags**:
  - Browser/wizard: `--browser` (bool, true; open loopback wizard, `=false` for TTY), `--no-browser` (bool, false; sugar for `--browser=false`), `--browser-addr` (string, ""; host:port, empty → 127.0.0.1:9100, or 0.0.0.0:9100 with `--public`), `--browser-open` (bool, true; launch system browser), `--ssh-tunnel` (bool, false; bind loopback + print `ssh -L` command), `--public` (bool, false; non-loopback bind + TLS + token auth), `--public-host` (string, ""; hostname/IP in URL), `--browser-idle-timeout` (duration, 0 → 30m), `--manage-firewall` (bool, true; open/close port in `--public`).
  - Lifecycle: `--non-interactive` (bool, false; no prompts), `--component` (string, ""; controller|daemon|dashboard), `--install-mode` (string, ""; native|compose), `--service-mode` (string, ""; legacy systemd registration), `--startup-validation-mode` (string, ""; legacy health-validate), `--boot-mode` (string, ""; auto-start on boot), `--start-mode` (string, ""; start now).
  - Controller: `--controller-install-dir` (default `/opt/prexorcloud/controller`), `--controller-mongo-mode` (local|remote), `--controller-mongo-uri`, `--controller-redis-mode`, `--controller-redis-uri`, `--controller-http-port`, `--controller-grpc-port`, `--controller-cors-origin`.
  - Daemon: `--daemon-install-dir` (default `/opt/prexorcloud/daemon`), `--daemon-node-id`, `--daemon-controller-host`, `--daemon-controller-grpc-port`, `--daemon-controller-http-port` (default 8080), `--daemon-join-token`.
  - Dashboard: `--dashboard-install-dir` (default `/opt/prexorcloud/dashboard`), `--dashboard-public-url`, `--dashboard-serve-mode` (default `nginx`), `--dashboard-tls-mode` (default `none`), `--dashboard-tls-email`, `--dashboard-controller-url`, `--dashboard-admin-user` (default `admin`), `--dashboard-admin-password`, `--dashboard-listen-port` (default `80`).
- **Does**: Default UI is a browser wizard on `127.0.0.1:9100` (`runBrowserSetup`), falling back to a TTY flow on headless hosts. Controller install resolves Mongo/Redis (local install or remote URI), downloads + cosign-verifies the release jar, writes `controller.yml`, configures the CLI context, and installs/starts via compose or a `prexorcloud-controller.service` systemd unit (with auto-login as `admin`). Daemon install downloads the daemon jar, writes `daemon.yml`, exchanges the join token to enrol, and installs via compose or `prexorcloud-daemon.service`. Dashboard install is compose-only via the CLI, writes an nginx compose project, registers the CORS origin, and saves a `dashboard` context.
- **JSON**: no.
- **Interactive**: full browser wizard by default; TTY path uses component/install-mode selection and per-component prompts.
- **Disposition**: CHANGE — collapse 4 lifecycle flags into 2 (deprecate --service-mode/--startup-validation-mode in favor of --boot-mode/--start-mode); remove dead --dashboard-serve-mode/--dashboard-tls-email; fix --component help to list 'dashboard'

- **Path**: `prexorctl status`
- **Purpose**: Cluster overview dashboard.
- **Args**: none.
- **Flags**: none beyond global.
- **Does**: `--json` → `GET /api/v1/overview` dumped raw. Non-JSON → fetches the overview plus `GET /api/v1/groups` and best-effort `GET /api/v1/system/version`, then renders a banner, a CLUSTER/NODES/INSTANCES summary, a live-metrics card, a groups table, and footer hints.
- **JSON**: partial — only the `/api/v1/overview` object (no groups/version).
- **Interactive**: none.
- **Disposition**: CHANGE — wire real metrics or remove the synthetic TPS/Players/Memory sparklines + group SPARK column; fix whoami='(authenticated)'

- **Path**: `prexorctl config view`
- **Purpose**: Show the active context's controller, masked token, accent, and config path.

### `prexorctl config set <key> <value>`
- **Path**: `prexorctl config set <key> <value>`
- **Purpose**: Set a value on the active context.
- **Args**: `<key>`, `<value>` (both required). Keys: `controller` (URL-validated), `token`, `accent` (purple|cyan|green|amber).
- **Flags**: none beyond global.
- **Does**: Writes the value to the active context, auto-creating a `default` context for controller/token.
- **JSON**: no.
- **Interactive**: none.
- **Disposition**: CHANGE — never take a secret/token as a positional arg (leaks to shell history); read from stdin

### `prexorctl config unset <key>`
- **Path**: `prexorctl config unset <key>`
- **Purpose**: Clear a value on the active context.
- **Args**: `<key>` (required): `controller`, `token`, or `accent`.
- **Flags**: none beyond global.
- **Does**: Removes the key from the active context.
- **JSON**: no.
- **Interactive**: none.

## Proposed commands & flags (not in the CLI today)

New surface drawn from the design-review's symmetry gaps and broader redesign notes (see `cli-design-review.md`).

### user
- `prexorctl user show <name>` and `prexorctl user update <name>` — asymmetric with `role`, which already has both.

### list parity
- Add `--filter` / `--sort` / `--watch` to `instance list` and `node list` (parity with `group list`, where they already exist).

### network
- A `prexorctl network ...` group — networks are REST-managed only today; there is no CLI surface for them.

### auth & identity
- `prexorctl whoami` / `prexorctl auth status` — current user, role, token expiry, active endpoint/leader.
- `prexorctl auth refresh` / `auth change-password` — token refresh + password change (server endpoints reportedly already exist; verify).
- `prexorctl config get <key>` — single-value accessor; `prexorctl context rename` and `context --discover` (fill endpoints from `/cluster/members`).

### cluster & ops
- `prexorctl cluster health` (fused verdict), `cluster leader` (show/transfer), `cluster leases` (route exists, no verb today).
- `prexorctl backup pull` / `restore --from-file` — the CLI cannot retrieve a bundle today.

### modules
- `prexorctl module build | sign | bundle` — the CLI consumes signed bundles but can't produce them.
- `prexorctl module enable | disable | describe`; `prexorctl module registry list | add | remove`.

### unified verbs (GitOps layer — see roadmap Phase 3/4)
- `prexorctl get <type>` / `describe <type> <name>` with resource aliases; `apply -f` / `diff` / `edit` / `explain`.
